Wprowadzenie

Celem projektu jest budowa modelu liniowego dla spalania samochodw. Model budowany jest na podstawie pliku cars.csv, ktry przedstawia spalanie, liczb cylindrw, objto silnika, moc w koniach mechanicznych, wag, czas przyspieszenia, rok i miejsce produkcji dla danych modeli samochodw. Dane s podanie w systemie amerykaskim, dlatego zachodzi bdzie konieczno przeksztacenia ich na system europejski.
UWAGA!Przeniosem formatowanie danych do kolejnej czci, aby we wprowadzeniu zaj si oglnym wprowadzeniem oraz przedstawieniem hipotez. Zdecydowaem si te na umieszczenie kodu rwnie w pliku html, eby nie byo potrzeby zagldania do pliku rdowego.
Hipotezy: Co do przedziau ufnoci, myl, e ten bootstrapowy bdzie wszy, dla wszystkich parametrw modelu bdzie mniejszy rozrzut. Spodziewam si te lepszego dopasowania do danych dla modelu bootstrapowego - bdzie za tym i wikszy wspczynnik determinacji, lepsze kryteria informacyjne. Parametry nie powinny si zbytnio rni (podejrzewam, e ten sam zbir zostanie wskazany w obu metodach).
Jeli chodzi o to, ktre zmienne znajd si w modelu, na samym wstpie mog odrzuci nazwy samochodw. Nie bdzie te na pewno albo pojemnoci silnika, albo liczby cylindrw (moe ich obu), gdy objto silnika zaley od liczby cylindrw. Nie wydaje mi si te, eby pochodzenie samochodu wpywao szczeglnie na spalanie.

Do znalezienia najlepszego zbioru posuy mi metoda Hellwiga, ktra uwzgldnia korelacj (oraz wspczynnik zmiennoci, to jednak zostao przeze mnie przeanalizowane osobno).

hellwig <- function( y, x, method="pearson")
{
  requireNamespace("utils")
  x <- as.data.frame(x)
  cm <- stats::cor(x, method=method) # korelacje wewntrz zmiennych niezalenych 
  cd <- stats::cor(x, y, method=method) # korelacje ze zmienn zalen
  # lista kombinacji zmiennych
  k <- sapply( seq(2, length(x)), function(i)
    utils::combn(length(x), i, simplify=FALSE) )
  k <- do.call("c", k)
  # liczenie indywidualnych wskanikw pojemnoci informacyjnych
  hfun <- function(v)
  {
    sapply(v, function(i) cd[i]^2 / sum(abs(cm[v,i])) )
  }
  #budowanie ramki danych dla zintegrowanych wskanikw pojemnoci informacyjnych
  h <- sapply(k, hfun)
  data.frame( k = sapply( k, paste, collapse="-"),
              h = sapply(h, sum),
              stringsAsFactors=FALSE)
}

Dla wskazanej kombinacji zbuduj model liniowy i sprawdz, czy nie ma w nim katalizatorw:

katalizator <- function(y, xlist){
  k = length(xlist)
  ramka <- data.frame(y,xlist)

  #tworz macierze R i R0
  R <- matrix(data=0,nrow = k, ncol = k)
  R0 <- matrix(data=0,nrow = k, ncol = 1)
  for (i in 1:k){
    COR <- cor(ramka$y,ramka[i+1])
    R0[i,1]=COR
    for (j in 1:k){
      COR <- cor(ramka[i+1],ramka[j+1])
      R[i,j]= COR
    }
  }

  #macierz "przejscie" pozwoli na stworzenie regularnego R0 poprzez okrelenie     kolejnoci zmiennych w regularnym R0
  przejscie <- matrix(data=0, nrow = k, ncol = 2)
  R0_regularne <- matrix(data=sort(abs(R0)),nrow = k,ncol = 1)
  for (i in 1:k){
    for (j in 1:k){
      if (R0_regularne[i,1] == abs(R0[j,1])){
        przejscie[i,1]=j
        if (R0[j,1]>0){
          przejscie[j,2]=1
        }
        else{
          przejscie[j,2]=-1
        }
      }
    }
  }
  
  #tworzenie regularnego R
  R_regularne <- R
  for (i in 1:k){
    i = przejscie[i,1]
    for (j in 1:k){
      R_regularne[i,j]=przejscie[przejscie[j,1],2]*przejscie[przejscie[i,1],2]*R[przejscie[j,1],przejscie[i,1]]
      R_regularne[j,i]=R_regularne[i,j] 
    }
  }
  
  katalizator <- 0
  
  #sprawdzanie macierzy R_regularne pod ktem obecnoci zmiennych-katalizatorW
  for (i in 1:k){
    for (j in 1:k){
      a <- R_regularne[i,j]
      if ((i<j) & (a < 0 | a > (R0_regularne[i,1]/R0_regularne[j,1]))){
        katalizator = a
      }
    }
  }
  zz <- 0

  for (i in 1:k){
    for (j in 1:k){
      numer <- przejscie[i,1]
      if ((R[i,j] == katalizator | abs(R[i,j]) == katalizator) & i<j){
        print("Numer zmiennej objaniajcej bdcej katalizatorem:")
        print(numer)
        zz = zz+1
        print("Warto katalizatora:")
        print(katalizator)
      }
    }
  }
  if (zz == 0){
    print("Brak katalizatorw")
  }
}

Na pierwszym wykresie, gdzie skonfrontowane s reszty modelu z jego dopasowaniem, mona zauway pask, czerwon lini, co wiadczy o spenieniu zaoenia dotyczcego liniowoci modelu. Punkty tworz chmur, nie wida, aby na pocztku czy kocu oddalay si od linii, co wiadczy o tym, e model nie jest heteroskedastyczny.
Na wykresie Q-Q (kwantyl-kwantyl) wida, e punkty tworz lini prost (z wyjtkiem samego pocztku i koca - moe by za mao danych w modelu), co wiadczy o tym, e spenione jest zaoenie o normalnoci reszt.
Na trzecim wykresie, Scale-Location, wida do pask, czerwon lini, co potwierdza homoskedastyczno modelu.
Na ostatnim z czterech wykresw, Residuals vs Leverage, przedstawiona pokazaa odlego Cooka, czyli miara stopnia zmiany wspczynnikw regresji, gdyby dany przypadek pomin w obliczeniach wspczynnikw (miara wpywu poszczeglnych obserwacji na prost regresji). Gdyby ktry przypadek znalaz si za lini Cooka, naleaoby go wykluczy z modelu, aby by on bardziej dopasowany. W tym przypadku jednak wszystko jest w porzdku, takie punkty nie istniej.

Poza skal, wykresy prezentuj si tak samo jak w poprzednim przypadku. Tu rwnie nie spenio si moje podejrzenie, ponownie wikszy rozrzut jest dla metody boootstrapowej.

Podsumowanie

Ostatecznie, w obu przypadkach, spalanie zaley od koni mechanicznych i wagi samochodu. Sprawdziy si moje hipotezy dotyczce braku zmiennej displacement lub/i cylinders oraz braku wpywu pochodzenia. Zostay spenione zaoenia dotyczce normalnoci rozkadu reszt, heteroskedastycznoci, liniowoci modelu oraz autokorelacji.
Model bootstrapowy okaza si by lepiej dopasowany do modelu, ale wiza si to moe z przeuczeniem modelu, cho rnica midzy dwoma modelami jest nieznaczna i moe wynika z wartoci parametrw.
Zaskoczenie stanowiy wykresy przedziaw ufnoci dla modeli - spodziewaem si odwrotnych rezultatw.
