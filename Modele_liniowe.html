Wprowadzenie

Celem projektu jest budowa modelu liniowego dla spalania samochodów. Model budowany jest na podstawie pliku “cars.csv”, który przedstawia spalanie, liczbę cylindrów, objętość silnika, moc w koniach mechanicznych, wagę, czas przyspieszenia, rok i miejsce produkcji dla danych modeli samochodów. Dane są podanie w systemie amerykańskim, dlatego zachodzić będzie konieczność przekształcenia ich na system europejski.
UWAGA!Przeniosłem formatowanie danych do kolejnej części, aby we wprowadzeniu zająć się ogólnym wprowadzeniem oraz przedstawieniem hipotez. Zdecydowałem się też na umieszczenie kodu również w pliku html, żeby nie było potrzeby zaglądania do pliku źródłowego.
Hipotezy: Co do przedziału ufności, myślę, że ten bootstrapowy będzie węższy, dla wszystkich parametrów modelu będzie mniejszy rozrzut. Spodziewam się też lepszego dopasowania do danych dla modelu bootstrapowego - będzie za tym iść większy współczynnik determinacji, lepsze kryteria informacyjne. Parametry nie powinny się zbytnio różnić (podejrzewam, że ten sam zbiór zostanie wskazany w obu metodach).
Jeśli chodzi o to, które zmienne znajdą się w modelu, na samym wstępie mogę odrzucić nazwy samochodów. Nie będzie też na pewno albo pojemności silnika, albo liczby cylindrów (może ich obu), gdyż objętość silnika zależy od liczby cylindrów. Nie wydaje mi się też, żeby pochodzenie samochodu wpływało szczególnie na spalanie.

Do znalezienia najlepszego zbioru posłuży mi metoda Hellwiga, która uwzględnia korelację (oraz współczynnik zmienności, to jednak zostało przeze mnie przeanalizowane osobno).

hellwig <- function( y, x, method="pearson")
{
  requireNamespace("utils")
  x <- as.data.frame(x)
  cm <- stats::cor(x, method=method) # korelacje wewnątrz zmiennych niezależnych 
  cd <- stats::cor(x, y, method=method) # korelacje ze zmienną zależną
  # lista kombinacji zmiennych
  k <- sapply( seq(2, length(x)), function(i)
    utils::combn(length(x), i, simplify=FALSE) )
  k <- do.call("c", k)
  # liczenie indywidualnych wskaźników pojemności informacyjnych
  hfun <- function(v)
  {
    sapply(v, function(i) cd[i]^2 / sum(abs(cm[v,i])) )
  }
  #budowanie ramki danych dla zintegrowanych wskaźników pojemności informacyjnych
  h <- sapply(k, hfun)
  data.frame( k = sapply( k, paste, collapse="-"),
              h = sapply(h, sum),
              stringsAsFactors=FALSE)
}

Dla wskazanej kombinacji zbuduję model liniowy i sprawdzę, czy nie ma w nim katalizatorów:

katalizator <- function(y, xlist){
  k = length(xlist)
  ramka <- data.frame(y,xlist)

  #tworzę macierze R i R0
  R <- matrix(data=0,nrow = k, ncol = k)
  R0 <- matrix(data=0,nrow = k, ncol = 1)
  for (i in 1:k){
    COR <- cor(ramka$y,ramka[i+1])
    R0[i,1]=COR
    for (j in 1:k){
      COR <- cor(ramka[i+1],ramka[j+1])
      R[i,j]= COR
    }
  }

  #macierz "przejscie" pozwoli na stworzenie regularnego R0 poprzez określenie     kolejności zmiennych w regularnym R0
  przejscie <- matrix(data=0, nrow = k, ncol = 2)
  R0_regularne <- matrix(data=sort(abs(R0)),nrow = k,ncol = 1)
  for (i in 1:k){
    for (j in 1:k){
      if (R0_regularne[i,1] == abs(R0[j,1])){
        przejscie[i,1]=j
        if (R0[j,1]>0){
          przejscie[j,2]=1
        }
        else{
          przejscie[j,2]=-1
        }
      }
    }
  }
  
  #tworzenie regularnego R
  R_regularne <- R
  for (i in 1:k){
    i = przejscie[i,1]
    for (j in 1:k){
      R_regularne[i,j]=przejscie[przejscie[j,1],2]*przejscie[przejscie[i,1],2]*R[przejscie[j,1],przejscie[i,1]]
      R_regularne[j,i]=R_regularne[i,j] 
    }
  }
  
  katalizator <- 0
  
  #sprawdzanie macierzy R_regularne pod kątem obecności zmiennych-katalizatoróW
  for (i in 1:k){
    for (j in 1:k){
      a <- R_regularne[i,j]
      if ((i<j) & (a < 0 | a > (R0_regularne[i,1]/R0_regularne[j,1]))){
        katalizator = a
      }
    }
  }
  zz <- 0

  for (i in 1:k){
    for (j in 1:k){
      numer <- przejscie[i,1]
      if ((R[i,j] == katalizator | abs(R[i,j]) == katalizator) & i<j){
        print("Numer zmiennej objaśniającej będącej katalizatorem:")
        print(numer)
        zz = zz+1
        print("Wartość katalizatora:")
        print(katalizator)
      }
    }
  }
  if (zz == 0){
    print("Brak katalizatorów")
  }
}

Na pierwszym wykresie, gdzie skonfrontowane są reszty modelu z jego dopasowaniem, można zauważyć płaską, czerwoną linię, co świadczy o spełnieniu założenia dotyczącego liniowości modelu. Punkty tworzą chmurę, nie widać, aby na początku czy końcu oddalały się od linii, co świadczy o tym, że model nie jest heteroskedastyczny.
Na wykresie Q-Q (kwantyl-kwantyl) widać, że punkty tworzą linię prostą (z wyjątkiem samego początku i końca - może być za mało danych w modelu), co świadczy o tym, że spełnione jest założenie o normalności reszt.
Na trzecim wykresie, Scale-Location, widać dość płaską, czerwoną linię, co potwierdza homoskedastyczność modelu.
Na ostatnim z czterech wykresów, Residuals vs Leverage, przedstawiona pokazała odległość Cooka, czyli miara stopnia zmiany współczynników regresji, gdyby dany przypadek pominąć w obliczeniach współczynników (miara wpływu poszczególnych obserwacji na prostą regresji). Gdyby któryś przypadek znalazł się za linią Cooka, należałoby go wykluczyć z modelu, aby był on bardziej dopasowany. W tym przypadku jednak wszystko jest w porządku, takie punkty nie istnieją.

Teraz zweryfikuję, które zmienne są statystycznie istotne. Powinno to przebiegać stopniowo, bez wyrzucania wszystkich zmiennych nieistotnych jednocześnie, jednak nie posiadam pętli, która by to wykonała. Jeśli chodzi o poprawność modelu, lepiej wypada model bootstrapowy. Współczynnik determinacji jest wyższy niż w modelu klasycznym, a skorygowane R^2 jest tak samo wysokie jak sam współczynnik (model nie jest ani trochę przeparametryzowany). Oba kryteria informacyjne również wypadają lepiej (bootstrap daje lepsze dopasowanie modelu, ale z większym prawdopodobieństwem jest przeuczony). Należy jednak zwrócić uwagę na to, że różnice te są nieduże, ale nie zmienia to faktu, że trafnie przewidziałem różnice w obu modelach.
Parametr przy zmiennej “Horsepower” jest większy niż w modelu klasycznym, natomiast przy parametrze “Weight” sytuacja jest odwrotna. Wykonam teraz wykres dla przedziałów ufności w bootstrapie i w modelu klasycznym, co pozwoli zobrazować różnicę.

coefs_model <- summary(model)$coefficients

set.seed(123)
wykres <- tibble(bootstrap = zbiorek,
                 classic = 
                   coefs_model[1,1] + 
                   rnorm(10^4)*coefs_model[1,2] ) %>%
  gather(type, parameter)


wykres %>%
  ggplot(aes(x = parameter, col = type)) + 
  geom_density()

Jak widać na wykresie, przedział ufności dla zmiennej “horsepower” jest węższy i wyższy przy metodzie klasycznej. Spodziewałem się sytuacji odwrotnej. Parametry dla zmiennej “horsepower” okazały się mieć większy “rozrzut” dla metody bootstrapowej.

set.seed(123)
wykres <- tibble(bootstrap = zbiorek2,
                 classic = 
                   coefs_model[2,1] + 
                   rnorm(10^4)*coefs_model[2,2] ) %>%
  gather(type, parameter)


wykres %>%
  ggplot(aes(x = parameter, col = type)) + 
  geom_density()

Poza skalą, wykresy prezentują się tak samo jak w poprzednim przypadku. Tu również nie spełniło się moje podejrzenie, ponownie większy “rozrzut” jest dla metody boootstrapowej.

Podsumowanie

Ostatecznie, w obu przypadkach, spalanie zależy od koni mechanicznych i wagi samochodu. Sprawdziły się moje hipotezy dotyczące braku zmiennej “displacement” lub/i “cylinders” oraz braku wpływu pochodzenia. Zostały spełnione założenia dotyczące normalności rozkładu reszt, heteroskedastyczności, liniowości modelu oraz autokorelacji.
Model bootstrapowy okazał się być lepiej dopasowany do modelu, ale wiązać się to może z przeuczeniem modelu, choć różnica między dwoma modelami jest nieznaczna i może wynikać z wartości parametrów.
Zaskoczenie stanowiły wykresy przedziałów ufności dla modeli - spodziewałem się odwrotnych rezultatów.
